## 补充说明
>要知道的是，地面站通过IP连上无人机后是没有任何数据交往的，官方的步骤太简略且没有统一说明

能进行数据交互的前提是让机载电脑（真机）或仿真系统中启动相应的通信节点：

1. 启动 **roscore**
```bash
roscore # 此命令将启动 ROS Master、ROS 参数服务器和 rosout 日志节点
```
2. 启动 **sunray_communication_bridge节点**
```bash
roslaunch sunray_communication_bridge sunray_communication_bridge.launch uav_id:="1" uav_experiment_num:="1"
```
**注意！！！ uav_id与uav_experiment_num是必须参数，uav_id设置无人机的ID，uav_experiment_num设置无人机的数量，其中还有一个重要的参数uav_name用于定义接收话题的前缀，默认值为uav。若前缀对不上px4发布的话题将无法向地面站发送数据**

3. 启动 **external_fusion节点**
```bash
roslaunch sunray_uav_control external_fusion.launch external_source:=2 # 此脚本会实时向外发送无人机相关状态数据
```
**此脚本发布的话题前缀默认为uav**

4. 启动 **sunray_control节点**
```bash
roslaunch sunray_uav_control sunray_control_node.launch # 用于接收外界传来的控制指令
```
## 调试指令
查看对应节点 **发布/订阅** 的话题
```bash
rosnode info 节点路径

# 例如查看sunray_communication_bridge订阅的话题
rosnode info /sunray_communication_bridge
```
